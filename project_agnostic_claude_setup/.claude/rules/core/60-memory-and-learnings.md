# Memory and Learnings

- Use `.claude/memory/checklist.md` and `.claude/memory/progress.md` only when work spans sessions or multiple coordinated tasks.
- Capture durable learnings only after a verified fix or a repeated pattern.
- Keep entries short: trigger, rule, evidence.
- Never store secrets, credentials, or machine-local paths in memory files.
- Do not let long-lived docs get deleted during cleanup.
